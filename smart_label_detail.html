智慧 OK 繃（Smart Band-aid）
Disposable Wound Monitor · 一次性傷口監測
醫療穿戴 一次性 溫度監測 感染預警
📌 痛點分析

術後傷口感染是醫療機構最常見的併發症之一，糖尿病足患者尤其高風險。傳統做法依賴護理人員定時巡檢翻開紗布觀察，不僅耗費人力，且發現感染時往往已錯過黃金介入時機。市面尚無低成本、一次性、可即時監測的解決方案。

💡 方案概述

HB120 內建 ±1°C 溫度感測器，嵌入一次性傷口敷料中。敷料貼附後，持續廣播傷口表面溫度數據。當溫度連續上升超過設定閾值（如 37.5°C），護理站或手機 APP 即時收到推送警報，在感染可視化前即介入處理。

🔧 技術實現
  • HB120 以 CR1220 鈕扣電池供電，持續廣播 >30 天
  • 敷料內嵌柔性 PCB，總厚度 <1.5mm，不影響傷口透氣
  • Data Logging 記錄每 5 分鐘一次溫度，回診時 BLE 讀取歷史曲線
  • 可選 NFC 標籤共存，手機貼近即可讀取患者 ID 與溫度趨勢
💰 商業價值
  • 減少不必要的換藥巡檢，釋放護理人力
  • 降低院內感染率，縮短平均住院天數
  • 糖尿病足患者全球約 5.3 億，一次性傷口監測需求龐大
  • 可延伸至運動傷害、嬰兒尿布疹監測等場景
🧑‍⚕️ 傷口敷料貼附 → 📡 HB120 持續廣播溫度 → ⚠️ 異常升溫即時警報 → 🏥 醫護提早介入

布巾追蹤標籤（Laundry Tag）
Hospitality Linen Tracking · 旅館布巾資產管理
旅館業 布巾管理 資產追蹤 工業洗滌
📌 痛點分析

大型旅館每月處理數十萬件布巾，遺失率高達 5–10%。傳統方式靠人工盤點，耗時且不精確。RFID 標籤加上讀取器基礎設施，對多數旅館部署門檻過高且複雜。

💡 方案概述

HB120 標籤縫入或熱壓在布巾角落，每個標籤帶有唯一 ID。布巾在洗滌、烘乾、折疊、配送各站點由 BLE 閘道器自動讀取，無需人工掃描。管理系統即時顯示每件布巾位置與洗滌次數，超限自動報廢汰換。

🔧 技術實現
  • 標籤封裝於耐高溫矽膠環中，可承受 90°C 工業洗滌、烘乾 (>5,000 次)
  • 每層樓走道部署 1–2 個 BLE 閘道器，覆蓋 300–500 件布巾
  • HB120 廣播間隔可設為 1–2 分鐘，鈕扣電池壽命 >2 年
  • Data Logging 記錄洗滌累計次數，支援預測性汰換排程
💰 商業價值
  • 布巾遺失率顯著降低，投資回收期短
  • 自動化盤點節省大量人力工時
  • 精準掌握布巾生命週期，優化汰換時機
  • 全球旅館布巾管理需求龐大，自動化滲透率仍低

智慧瓶蓋（Smart Bottle Cap）
Beverage Temperature · 飲料溫度即時顯示
食品飲料 超商 溫度標籤 消費者互動
📌 痛點分析

超商熱飲（關東煮、咖啡、熱茶）的溫度是食安與口感的關鍵，但消費者無法得知實際溫度 — 太燙燙口、太冷失去風味。業者也無法有效追蹤商品在貨架上的溫度變化，尤其是冬季熱飲保溫與夏季冷飲的食安管理存在盲區。

💡 方案概述

HB120 嵌入飲料瓶蓋內側，持續感測飲料溫度並以 BLE 廣播。消費者手機靠近即可看到即時溫度及「最佳飲用」提示（如 45–55°C 適合熱咖啡、<5°C 適合冷飲）。開瓶時自動偵測並記錄開瓶時刻，可推播賞味期限提醒。

🔧 技術實現
  • HB120 內建溫度感測 ±1°C 精準度，直接接觸飲料
  • 瓶蓋內建薄型 CR1220 電池，持續廣播 >60 天（出廠到消費）
  • 開瓶偵測透過電容式感應或磁簧開關
  • 可選 Data Logging 記錄整條冷鏈溫度曲線，食安稽核用
💰 商業價值
  • 品牌加值行銷新載體，掃描瓶蓋即進入品牌 APP
  • 消費者互動率提升，精準投放優惠與資訊
  • 冷鏈異常即時通報，減少食安客訴與退貨
  • 日本超商每年銷售 >200 億瓶飲料，導入空間巨大
🧴 瓶蓋鎖定啟動 → 📡 持續溫度廣播 → 📱 消費者掃描查看 → 🎁 品牌互動 + 優惠

校園安全標籤（School Safety Tag）
Student Location · 學童定位與安全預警
教育 兒童安全 定位 批量部署
📌 痛點分析

學童安全是家長與學校最關心的議題。遠足、校外教學時，老師須手動點名確認學生人數，耗時且容易遺漏。放學後若學生未按時返家，家長無法即時得知。現有 GPS 手錶成本高、需充電、且容易被學生取下，不適合小學生批量部署。

💡 方案概述

HB100 標籤嵌入學童書包或名牌，無需充電、無需操作。學校出入口部署 BLE 閘道器，學生通過時自動感測並上傳名單。遠足時老師手機 APP 可即時顯示所有學生位置。放學後若學生在校異常滯留超過設定時間，系統自動推播通知家長。

🔧 技術實現
  • HB100 以 CR2032 供電，持續廣播 >1 年（每日 8 小時）
  • 閘道器覆蓋半徑 30–50m，校門口 1–2 台即可
  • 標籤 ID 綁定學生資料，可選防拆偵測（剪斷即觸發警報）
  • 每學年更換新標籤，舊標籤可回收
💰 商業價值
  • 每位學童年負擔極低，可規模化部署
  • 台灣小學生約 120 萬人，日本約 600 萬人，市場規模巨大
  • 可擴展至幼兒園、安親班、特殊教育機構
  • 政府標案 / 校園安全預算支持，B2G 模式可行
🎒 書包內嵌標籤 → 🚪 出入校自動感測 → 📱 家長 APP 即時通知 → ⚠️ 異常滯留預警

臨床試驗遵從性（Clinical Trial Compliance）
Smart Pill Label · 智慧藥瓶標籤
製藥 臨床試驗 Data Logging 合規性
📌 痛點分析

臨床試驗中，患者服藥遵從性與藥物儲存條件是影響數據品質的兩大關鍵因素。傳統靠患者日誌自行記錄，不可靠且無法驗證。電子藥盒體積大、不適合大規模或多中心試驗。溫度敏感藥物若儲存不當，直接影響療效與試驗結果。

💡 方案概述

HB120 以貼紙形式貼附於藥瓶標籤層，每次開瓶自動記錄時間戳，同時持續記錄環境溫度（Data Logging）。回診時研究人員以手機或平板 BLE 讀取數據，無需取出標籤。所有數據加密儲存，符合 HIPAA / 個資法要求。

🔧 技術實現
  • HB120 標籤總厚度 <0.5mm，可整合於現有藥瓶標籤製程
  • Data Logging 可記錄 >6 個月溫度（每 10 分鐘一筆）
  • 開瓶偵測透過光感應或磁簧開關，記錄每次開瓶時間
  • BLE 讀取距離 1–2m，支援批量讀取（一次讀取整箱藥瓶）
💰 商業價值
  • 相較傳統方案，具備顯著成本優勢
  • 提升試驗數據完整性與可信度
  • FDA / PMDA 對數位化遵從性證據接受度提高，合規優勢明顯
  • 全球臨床試驗市場龐大，IoT 遵從性方案滲透率仍低
💊 藥瓶貼標 → 📝 每次開瓶自動記錄 → 🌡️ 持續溫度監測 → 📋 回診 BLE 讀取數據

智慧垃圾桶（Smart Waste Bin）
Fill-level Monitoring · 滿度監測與清運優化
智慧城市 環工 物聯網 節能減碳
📌 痛點分析

城市公共垃圾桶的清運多採用固定路線與固定頻率，常見「滿了無人收、空的卻去收」的無效率狀況。傳統超音波或紅外線感測方案需佈線或更換專用桶，成本高、維護難。清潔業者每年油耗與人力成本因無效巡檢浪費 30% 以上。

💡 方案概述

HB100 標籤貼附於桶蓋內側，搭配微型反射式紅外線感測器，定時量測桶內垃圾高度。當滿度超過 80% 時自動上傳警報至雲端平台。清運業者根據即時滿度數據規劃每日動態路線，優先處理已滿桶位，跳過未滿桶位。

🔧 技術實現
  • HB100 + 紅外線感測器，低 BOM 設計
  • 每日回報 4–6 次，鈕扣電池壽命 >18 個月
  • IP67 防水封裝，耐受日曬雨淋
  • 支援 LoRa / NB-IoT 閘道器回傳（視部署環境選用）
💰 商業價值
  • 每桶改造費用極低，對比傳統方案優勢明顯
  • 減少清運里程，節省油耗與碳排放
  • 日本台灣合計 >500 萬個公共垃圾桶，滲透空間大
  • 可延伸至回收分類桶、廚餘桶、大型垃圾子車
🗑️ 垃圾桶貼標 → 📡 即時滿度回報 → 🗺️ 動態路線規劃 → ⛽ 節省油耗 + 人力

鞋類生產追蹤（Shoe Production Tag）
Manufacturing Traceability · 產線即時可視化
製造業 鞋業 產線追蹤 可回收
📌 痛點分析

鞋類生產（尤其是運動鞋）涉及裁切、針車、成型、組裝、品檢等多道工序，傳統條碼掃描追蹤需要工作站逐一掃碼，耗時且容易漏掃。產線瓶頸難以即時掌握，品管追溯需翻找紙本紀錄效率低落。產能利用率普遍僅 60–75%。

💡 方案概述

每雙鞋在裁切階段即嵌入 HB100 標籤（放置於鞋舌或鞋墊下），BLE 主動廣播取代條碼掃描。各工作站安裝 BLE 閘道器，鞋子進入該站時自動被讀取，系統即時顯示在製品位置與各站加工時間。完工出貨前取出標籤，回收重複使用。

🔧 技術實現
  • HB100 以可換式電池供電，標籤可重複使用 >100 次
  • 閘道器每站 1 台，覆蓋整條產線
  • 廣播間隔 1 秒，確保鞋子在傳送帶上不被漏讀
  • 支援與 MES / ERP 系統即時對接（API / MQTT）
💰 商業價值
  • 部署與維護成本極具競爭力
  • 產能利用率提升，瓶頸站即時可視化
  • 品管追溯時間從數小時縮短至數秒
  • 全球製造業自動化追蹤需求強勁，鞋類年產量巨大

展會互動標籤（Event Interactive Tag）
Exhibition & Conference · 人潮分析與精準互動
展會活動 互動行銷 人潮分析 一次性
📌 痛點分析

展會主辦方難以量化各展位人潮，參展商無法判斷哪些參觀者真正有興趣。現有 RFID/NFC 方案需要參觀者主動排隊感應，互動率低。展後數據分析耗時且不精確，無法即時調整現場動線或攤位配置。

💡 方案概述

入場證或名片內建 HB100 標籤，參觀者靠近展位（<5m）時自動被展位 BLE 閘道器偵測。系統可自動推送展位資料、優惠券或問卷至參觀者手機。主辦方後台即時顯示展會熱區圖，參展商可獲得經過停留時間等精準訪客數據。

🔧 技術實現
  • HB100 標籤封裝於名片或入場證卡層，厚度 <0.8mm
  • 展位閘道器可設定偵測半徑（3–8m），避免跨展位干擾
  • 標籤 2–3 天活動電力足夠，活動結束可回收（資料清除）
  • 可選搭配 iBeacon 協議，手機 APP 在背景下自動觸發
💰 商業價值
  • 展位部署門檻極低，快速導入
  • 主辦方可提供全新數據產品（人潮熱圖、停留分析、展商 ROI）
  • 日本每年 >1,000 場大型展會（東京 Big Sight、幕張等），市場明確
  • 參展商願為精準潛在客戶數據付費，形成 SaaS 收入模式
🎪 入場證內嵌標籤 → 📡 靠近展位自動偵測 → 📱 即時互動觸發 → 📊 展會熱圖分析
